# 0000-需求规格说明书 (Requirements Specification)

## 1. 项目概览

**项目名称**: Lark CLI  
**定位**: 专为 AI Agent 和自动化场景设计的飞书/Lark 开放平台命令行工具。  
**语言要求**: 文档、注释、提交信息及交互均需使用中文。

## 2. 项目目标

本项目旨在对飞书官方 SDK (`@larksuiteoapi/node-sdk`) 进行轻量级 CLI 封装，使用户能够通过命令行直接调用飞书 OpenAPI 能力。

**核心解决的问题**:
- **降低门槛**: 开发者无需编写代码即可快速验证和使用飞书 API。
- **自动化集成**: 方便 Shell 脚本、CI/CD 流程及 AI Agent 直接调用飞书能力。
- **调试便捷**: 提供直观的命令行交互和详细的日志输出，便于排查 API 调用问题。

**主要适用场景**:
- 自动化运维脚本（如定期发送通知、同步数据）。
- AI 智能体（Agent）作为工具调用飞书 API。
- 快速原型开发与 API 功能验证。

## 3. 核心功能需求

### 3.1 认证管理 (Auth)
- **现状保持**: 继续支持 `App ID/Secret` + `User Token` (OAuth2) 模式。
- **单环境**: 暂不需要多环境/多租户切换功能，当前 `config.json` 机制满足需求。

### 3.2 消息模块 (Message)
- **消息类型**: 保持现有支持（文本、富文本、图片），暂不需要支持复杂的 JSON 交互式卡片。
- **功能**: 发送 (`send`)、历史记录 (`list`)、回复 (`reply`)。

### 3.3 文档与知识库 (Doc/Wiki) [重点增强]
- **Markdown 双向转换**:
  - `doc import`: 将本地 Markdown 文件（及图片）转换为飞书在线文档。
  - `doc export`: 将飞书在线文档下载为本地 Markdown 文件（及图片资源）。
- **基础功能**: 创建文档 (`create`)、获取纯文本 (`raw-content`)。

### 3.4 多维表格 (Base) [重点增强]
- **基础操作**: 列表 (`list`)、数据表 (`table`)、记录增删改查 (`record`)。
- **高级筛选**: 支持通过 JSON/DSL 进行复杂条件筛选（如公式、视图过滤）。
- **结构管理 (Meta)**: 
  - 支持创建新的多维表格应用。
  - 支持管理数据表字段（Field）、视图（View）的增删改。
- **自动化流程 (Automation)**:
  - 支持管理多维表格内部的自动化工作流（Automation/Flow）。

### 3.5 协作与效率
- **日历 (Calendar)**: 日程管理、忙闲查询。
- **任务 (Task)**: 任务创建、列表、完成。
- **电子表格 (Sheet)**: 基础创建与元数据查询。
- **云空间 (Drive)**: 文件上传、下载、列表、搜索。

## 4. 非功能性需求

### 4.1 交互与输出
- **输出格式**: 保持 `--json` 选项，输出完整 API 响应，供 AI Agent 解析。
- **日志调试**: 增加 `--verbose` 全局选项，输出详细的请求（URL, Headers, Body）和响应日志，便于排查问题。

### 4.2 性能与稳定性
- **Token 管理**: 自动处理 Token 过期与刷新 (`withAuthRetry`)。
- **错误处理**: 标准化错误码输出，结合 `--verbose` 提供更多上下文。
